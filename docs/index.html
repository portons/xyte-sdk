What is XYTE SDK?

Xyte is an IoT device management platform. The XYTE SDK gives you programmatic access to the entire platform — devices, spaces, incidents, tickets, and fleet analytics — through a type-safe TypeScript client, a deterministic CLI, and an interactive terminal UI.

For Operators

Interactive TUI with 8 screens. Inspect fleets, triage incidents, manage tickets, and generate branded PDF reports. Setup gating ensures readiness before operations.

For Developers

TypeScript SDK with typed namespaces: client.device.*, client.organization.*, client.partner.*. Call any endpoint safely with built-in guards for writes and deletes.

For AI Agents

The SKILL router gives agents a deterministic 5-step execution order with a 3-tier safety model. Headless JSON, MCP tools, and envelope outputs are all agent-native and schema-validated.

Features

Everything you need to operate device fleets at scale — from a single CLI.

Multi-Tenant Profiles

Switch between tenants and named API key slots. Keys stored in macOS Keychain or Linux secret-tool.

xyte tenant use acme && xyte auth key use --slot primary

Interactive TUI

8 operational screens: setup, config, dashboard, spaces, devices, incidents, tickets, and copilot. Full keyboard navigation.

xyte tui --tenant acme

Headless Mode

JSON-only NDJSON frames for agent consumption. Schema-versioned contracts. One-shot or streaming with reconnect metadata.

xyte tui --headless --screen dashboard --format json --once

Fleet Analytics

inspect fleet for health summaries. inspect deep-dive for 24h/72h/168h windows. Generate branded PDF reports.

xyte report generate --input deep-dive.json --out report.pdf

MCP Server

xyte mcp serve exposes 7 structured tools for external agent frameworks: setup, doctor, endpoints, call, inspect, report.

xyte mcp serve

Guarded Mutations

Read-only by default. --allow-write for mutations. --confirm <key> for destructive operations. Three-tier safety.

Read → --allow-write → --confirm <key>

Copy & Paste Workflows

Production-safe flows for operators and AI agents. Replace tenant IDs and keys with your own.

Operator Workflow Human 
# Install and onboard
npm install && npm run build
xyte

# Inspect fleet health
xyte inspect fleet --tenant acme --format ascii

# Generate PDF report
xyte inspect deep-dive --tenant acme --window 24 --format json > /tmp/dd.json
xyte report generate --input /tmp/dd.json --out /tmp/report.pdf

# Open interactive dashboard
xyte tui --tenant acme
Agent Workflow Machine 
# Verify readiness
xyte setup status --tenant acme --format json
xyte config doctor --tenant acme --format json

# Call endpoint with envelope
xyte call organization.devices.getDevices \
  --tenant acme --output-mode envelope --strict-json

# Headless snapshot
xyte tui --headless --screen dashboard --format json --once

# Start MCP server
xyte mcp serve

TypeScript SDK

Type-safe client with namespaced endpoints and a generic call method.

sdk-example.ts
import { createXyteClient } from '@xyte/sdk';

const client = createXyteClient();

// Typed namespace access
const devices = await client.organization.getDevices({ tenantId: 'acme' });

// Generic endpoint call
const tickets = await client.call('partner.tickets.getTickets', { tenantId: 'acme' });

Agent Skill Router

One file that tells any AI agent exactly how to operate your fleet — deterministic steps, safety tiers, and copy-paste commands.

What the SKILL file provides

  1. Setup & readiness gate
  2. Auth & key slot selection
  3. Endpoint operations (guarded)
  4. Fleet insights & reports
  5. Headless frames / MCP server

3-Tier Safety Model

READ (default) → WRITE (--allow-write) → DELETE (--confirm <key>)

Use Cases

How teams use the XYTE SDK in production.

MSP / NOC Daily Operations

  • Run inspect fleet and deep-dive for tenant health
  • Generate daily 24h/72h PDF reports for operations reviews
  • Use guarded endpoint calls for controlled remediation

Manufacturer Reliability

  • Track incident concentration by model/device over time windows
  • Use data-quality signals to identify telemetry inconsistencies
  • Send targeted commands with write guards for safe remote actions

Enterprise AV/IT Teams

  • Monitor spaces, devices, incidents, and tickets from TUI dashboard
  • Correlate active incidents with open tickets to prioritize resolution
  • Export PDF snapshots for operations and management reporting

AI Agent Integration

  • Use the SKILL router as the agent-native entry point for the entire workflow
  • Consume JSON-only headless frames and envelope outputs deterministically
  • Validate payloads against JSON schemas in CI before production loops
  • Run MCP tool server for structured tool calls from agent frameworks
